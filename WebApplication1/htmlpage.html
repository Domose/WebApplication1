Application Virtualization

App virtualization (application virtualization) is the isolation of an application from the end user's device that is used to access the software.
The application operates from the user's viewpoint just as it would if it existed on the local computer of the user. The user can shift or adjust
the application window size, and perform the operations of the keyboard and mouse. Often there may be slight variations(difference) however,
the consumer will usually have a smooth experience.

Application virtualization, also known as application service virtualization, is a term under the larger umbrella of virtualization. It refers
to executing an application or software on a thin client; a terminal or network workstation with few resident programs and most programs that
exist on a connected server is accessed. The client runs in an environment isolated from the operating system where the program is installed,
often referred to be encapsulated from the operating system that holds the application.

Application virtualization tricks a computer into functioning as if the program is running on the local machine, when in reality it's running on
a virtual machine (usually a server) situated in another location, using its operating system (OS), and is accessed by the local machine’s
operating system. Problems of incompatibility with the operating System of the local computer, or even bugs or low-quality code inside the
program, can be solved by running virtual applications.



How Application Virtualization Works

While there are several ways to virtualize software, IT teams mostly follow a server-based strategy, delivering the software without having to
install them on individual desktops. Instead, administrators deploy remote applications in the data centre on a server of an organization or
through a hosting service on a cloud and then distribute them to the user's computer.

To make this a reality, the IT teams must use an application virtualization product. Virtualization software vendors and their
products include Microsoft App-V, Citrix XenApp, Parallels Remote Application Servers, and VMware ThinApp or App Volumes- both of which
are included with VMware Horizon View. VMware also provides Horizon Applications to enable application virtualisation.



Types of Application Virtualization


✔Remote Application Virtualization- Remote applications run on a server, and end users can see and engage with their applications
on a network via remote control protocol. Remote programs can be fully integrated with the desktop of the user so that they appear
and behave as local applications, using technology known as seamless windows. To manage and execute remote applications
efficiently a constant network connectivity must be maintained.

✔Streaming Application Virtualization-The virtualized software is implemented on the local computer of the end-users during
application streaming. When an application is requested, its components are immediately retrieved on demand from the local
computer. In order to launch certain parts of the application are required, the rest can be downloaded as needed in the background.
Upon completion of the download process, a streaming application can run without any network connectivity.



Benefits of App Virtualization

1. App Management: Application virtualization makes the management and maintenance of applications in an enterprise much
simpler for IT departments. Instead of installing apps manually on every user's computer, App virtualization allows IT administrators
to install softwares once on a central server and then distribute the application on user devices as necessary. This also makes it easier
to upgrade or patch applications aside from reducing installation time, as IT just has to do so on a single server.

2. Scalability: Application virtualization allows IT administrators to distribute virtual applications to all sorts of connected devices,
irrespective of the operating systems or storage space used by such devices. This allows for thin client load balancing whereby users
access an application on a low-cost machine while centralized servers manage all the computing power required to run the application.
As a result, the company spends much less on computer resources as only basic computers are needed for employees to access the
apps they need for work. App virtualization also enables users to access applications that wouldn't usually execute on the operating system
of their computers, as the program actually runs on the centralized server. This is widely used to run a Windows program
directly on a Linux operating system.

3. Security: Application virtualization software gives IT administrators central control over which applications users can access. If the
application permissions of a user inside an organization changes, the IT administrator would simply restrict access of that user to an
application. Without application virtualization of the app, the IT administrator will have to uninstall the application physically from
the user's computer. This central control of access to apps is especially useful if a user's devices are stolen or lost, because the IT
administrator may revoke remote access to sensitive data without having to track the missing device.

4. Simplified OS deployment: Deploying a new OS within your network does not hinder the applications. Also, if you need to
reinstall an image on a desktop, you don't have to care about the programs running on this computer, as end users can access them on
the server.

5. Operating system independent: Often, virtualized applications are independent of OS. If you have Vista incompatible apps then
app virtualization of the program could be your solution. Virtualization of software paves the way for Vista x64 too. In a virtual world,
legacy applications which wouldn't work on a 64-bit device could run without problems on Vista x64. You can also run complex
Windows apps on Linux and OS X alongside Wine and Crossover.
